A desktop newsroom for local civic reporting

The Civic Desk helps small local publishers find sources, spot leads, draft and verify stories, and publish a reader-facing civic paper. It keeps the newsroom workspace local by default, while still supporting web sources and optional publishing providers.

Built For Small Local Newsrooms

Enough structure to help a part-time editor move faster, without taking judgment away from the editor.

Source Intake

Add sources manually, discover candidates, or import CSV, TXT, XLSX, DOCX, and text-readable PDF source lists. Imported and discovered sources go through review instead of being silently trusted.

Daily Scan

Fetches configured sources, records evidence, detects civic observations, and creates leads. Local AI can help summarize and rank, but deterministic source work happens first.

Dark Signal Desk

Surfaces public social, community, and weak-signal leads for human review. The app can rank and warn, but it must never hide information or veto an editor.

Advisor, Not Gatekeeper

Optional press-freedom and legal-risk checks can flag attribution, sourcing, privacy, and defamation concerns. They are advisory only; publication remains a human editorial decision.

How The Workflow Fits Together

Find, verify, write, approve, publish, and share.

1

Add and Review Sources

Start with official local sources, add local media and public community sources, then use search fallback to avoid missing useful leads.

2

Scan and Build Leads

The app records evidence, extracts entities, detects changes, and creates source-backed leads and dark-signal items for review.

3

Write and Edit

Use the Workbench to draft, revise, hold, approve, or send stories back for more reporting. Guardrails warn; editors decide.

4

Publish and Distribute

Compile a static issue, export a ZIP, publish with a provider, and generate share copy for newsletters and community platforms.

Publishing Stack

The output is a website package first. Distribution comes after the canonical paper exists.

here.now

Recommended default for simple temporary publication and cleanroom testing.

GitHub Pages

Recommended durable archive path for users who want a public repository-backed record.

Cloudflare and Netlify

More technical hosting options for users with provider accounts and tokens.

Substack and Newsletter

Generate copy-ready posts and newsletter markdown. Substack is an assisted workflow, not the only publishing route.

Architecture In Plain English

Your drafts and database are local unless you choose to fetch sources, connect a provider, export files, or publish.

Desktop app, local database, optional local AI

The app is a Tauri desktop app with a React interface, Rust backend, SQLite database, and Ollama local AI integration. The browser extension pairs through a loopback server on 127.0.0.1.

  • Local drafts and SQLite database.
  • Optional local model setup through Ollama.
  • Public web access for source fetches and publishing providers.
  • Static HTML output for long-term portability.
